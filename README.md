# BI-SU Member's App

BI-SU（エムスタイルジャパンが運営する、人向けのツバメの巣 健康食品・化粧品ブランド）の会員向けスマホアプリ開発プロジェクト。

- 対象プラットフォーム: iOS / Android（Flutter）
- 前段で HTMLモックアップ・要件定義書・仕様書を作成済み（下記「関連資産」）

## このリポジトリの現在の内容

AI駆動開発プログラム（平川さん）の課題に基づく、**ユーザーストーリーと受け入れ条件（DoD）**のアウトプット。

```
docs/
└── user-stories/
    ├── 00-overview.md            # ペルソナ・前提・共通DoD・ストーリー一覧
    ├── epic1-membership-card.md  # Epic: 会員証
    ├── epic2-stores.md           # Epic: 店舗
    ├── epic3-subscription.md     # Epic: 定期情報
    ├── epic4-purchase-history.md # Epic: 購入情報
    └── epic5-settings.md         # Epic: 設定
```

## 作成プロセス（課題のコツへの対応）

1. **目的指向**: 「実装計画に使えるユーザーストーリー集（受け入れ条件付き）を完成させる」をゴールに設定して作業（Claude Code の /goal 運用）
2. **ドラフト作成**: 確定済みの要件定義書 v1.3・仕様書 v1.3 の決定事項（閲覧専用・店舗POS=スマレジ・前年確定ステータス表示など）を反映してストーリー化
3. **マルチエージェント精査**: 独立した2つのレビューエージェント（プロダクトオーナー視点 / 抜け漏れ・整合性視点）で精査し、指摘を反映

## 課題の原文（2026-06-12 平川さん）

> 1. 各自今回のプロジェクト用のGitリポジトリをつくる
> 2. 作りたいアプリのユーザーストーリーと受け入れ条件までをMDファイルにアウトプットさせる（タスクまではアウトプットしなくていいです。）
> コツ：①/goalコマンドなどで目的指向にすること。②アウトプットに対してadvisorやマルチエージェントに抜け漏れを精査させること。
> 3. それをGitにあげる。
>
> ※Epicは下記とし、ユーザーストーリーを考えてしてください。
> 各ユーザーストーリーには受け入れ条件（DoD）を記載し、MDファイルにアウトプットすること
> ・会員証 ・店舗 ・定期情報 ・購入情報 ・設定（アナツバメについて/PUSH通知配信設定/メアド変更/住所変更/お名前変更など）
