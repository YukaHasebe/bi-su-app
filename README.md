## 画面構成（28画面以上）

```
BI-SU 会員アプリ
├── 認証・起動（SCR-0xx）
│   ├── SCR-001  スプラッシュ

│   ├── SCR-003  オンボーディング（初回のみ）

│   └── SCR-011  認証コード入力
│
├── [Tab 1] ホーム（SCR-1xx）
│   ├── SCR-100  ホーム / ステータス画面
│   │   ├── ランク別背景・3Dロゴエンブレム
│   │   ├── ポイント残高・進捗バー・年間購入累計

│   └── SCR-101  バーコード表示サブ画面

│
├── [Tab 2] 定期情報（SCR-2xx）
│   ├── SCR-200  定期情報一覧
│   └── SCR-201  定期詳細 → EXT-001（ECマイページ）
│
├── [Tab 3] 購入履歴（SCR-3xx）

│   └── SCR-301  注文詳細
│
├── [Tab 4] ストアーズ（SCR-4xx）
│   ├── SCR-400  店舗一覧（リスト⇔マップ切替）
│   └── SCR-401  店舗詳細 → EXT-003（地図アプリ）

│   ├── SCR-500  設定メニュー（8項目＋アカウント削除）
```

### 画面ID命名ルール

| ルール | 説明 |
|---|---|
| 百の位 = フッタータブ番号 | 1=ホーム, 2=定期情報, 3=購入履歴, 4=ストアーズ, 5=設定 |
| x00 | タブのルート画面 |
| x01以降 | サブ画面 |
| 0xx | 認証・起動フロー（タブ到達前） |
| DLG-xxx | ダイアログ / モーダル |

## 主要システム構成

### ハイブリッド方式（v2.1決定）

| 利用者 | ログイン | 利用可能な機能 |
|---|---|---|
| 未登録ユーザー（非会員） | 不要 | 仮会員証・店舗ポイント・チェックイン・ストアーズ・ブランドコンテンツ・多言語切替・問い合わせ |
| BI-SU会員（EC購入者・店舗来店者） | 必要 | 上記すべて＋定期情報・購入履歴・ランク制度・通知設定・ポイント優遇率 |

### 認証フロー（MIRANEST App準拠）

```
メールアドレス / 電話番号
  → 認証コード（メール / SMS）
    → 認証完了
      → 楽楽EC顧客データと照合（メールアドレス）
        → 一致: データ紐付け
        → 未登録: カスタマークリエイトAPIで顧客新規作成
```

### 会員ランク制度（5段階）

| ランク | 年間購入金額 | ポイント換算率 | 公式カラー |
|---|---|---|---|
| **MEMBER** | -- | 100円=1pt | シャンパン系 |
| SILVER | 30万円以上 | 100円=2pt | シルバー |
| GOLD | 50万円以上 | 100円=3pt | ゴールド |
| PLATINUM | 100万円以上 | 100円=5pt | 青系 |
| DIAMOND | 200万円以上 | 100円=10pt | 深紅系 |

- 集計期間: 当年1/1 -- 12/31の購入累計額
- 適用期間: 翌年1/1 -- 12/31（毎年再判定）
- アプリ表示: 前年確定の公式ステータス

### ポイント制度（2系統）

| 系統 | 管理元 | ログイン | 備考 |
|---|---|---|---|
| ECポイント（BASE） | 楽楽EC | 必要 | 既存制度。ランク連動で換算率が変動 |
| 店舗ポイント | アプリ独自 | 不要 | v2.1新設。来店購入・チェックインで付与。仮会員証でも利用可能 |

## 表記ルール

- 「会員」を使用（「会員様」「メンバー」は使わない）
  - 会員ランク階層名としての **Member / Gold / Platinum** 等の英語表記はOK
- 「MIRANEST商品」「BI-SU商品」の表記を使用（「製品」「プロダクト」「商品」単体は使わない）
- **MIRANEST Member's Club**（アポストロフィあり・英字表記固定）
- BI-SU に「（美巣）」は付けない
- 愛犬・愛猫で表現（猫を忘れない）
