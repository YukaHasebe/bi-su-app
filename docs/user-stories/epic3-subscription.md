# Epic 3: 定期情報

> 定期コースの「見える化」。アプリは閲覧専用（確定済み）のため、変更操作はECマイページへ確実に橋渡しする。EC顧客のユーザビリティ向上＝アプリの主目的のひとつ。

---

## US-3-01 次回お届けを確認する

> **会員**として、次の定期便がいつ届くかをすぐ確認したい。それは受け取りの予定を立て、飲み忘れなく続けたいからだ。

**受け入れ条件（DoD）**
- [ ] 利用中の各定期コースに、次回お届け予定日が表示される
- [ ] お届けサイクル（30日ごと等）が表示される
- [ ] 表示はECシステムの最新データと一致する（※依存: OP-24 EC API。データの更新タイミングが画面で確認できる）

## US-3-02 定期コースの内容を確認する

> **会員**として、自分がどの定期コースを、いくらで、何個受け取っているか確認したい。それは家計の把握と、コース内容の思い違い防止のためだ。

**受け入れ条件（DoD）**
- [ ] 各コースに商品画像・商品名・個数・金額（税込）が表示される
- [ ] 複数の定期コースを契約している場合、すべて一覧で確認できる
- [ ] ステータス（継続中など）がひと目で分かる
- [ ] 定期コースが0件の場合、空状態の案内と定期コースの紹介（ECへの導線）が表示される

## US-3-03 定期の変更手続きへ進む

> **会員**として、お届け日の変更やスキップをしたいとき、アプリから迷わず手続きに進みたい。それは電話をかけずに自分のタイミングで済ませたいからだ。

**受け入れ条件（DoD）**
- [ ] 各定期コースから「変更・スキップ・解約のお手続き」への導線があり、ECマイページの該当ページへ遷移できる（遷移体験は共通DoD参照）
- [ ] 次回お届けの変更締切日（何日前まで変更可能か）が表示される（※依存: EC仕様）
- [ ] 手続き後にアプリへ戻ると、変更が反映された表示になる

## US-3-04 解約済みの定期を確認する

> **会員**として、以前利用していた定期コースを確認したい。それは「また再開しようか」を考える材料にしたいからだ。

**受け入れ条件（DoD）**
- [ ] 解約済みコースが利用中と区別されて表示される（最終お届け日付き）
- [ ] 解約後12ヶ月以内のコースが表示される（※要決定: 表示期間。候補=6/12/24ヶ月）
- [ ] 解約済みコースから商品の再開・再購入導線（ECへの遷移）がある

## US-3-05 定期の異常（決済不備等）に気づく

> **会員**として、定期便が止まりそうなとき（カード期限切れ等）に早く気づきたい。それは「届くはずのものが届かない」を避けたいからだ。

**受け入れ条件（DoD）**
- [ ] 決済エラーや一時停止などの異常状態が、通常の「継続中」と明確に区別して表示される（※依存: OP-24 ECの定期ステータス仕様）
- [ ] 異常がある場合、ホームと定期情報タブにアラートバッジが表示され、該当コースが一覧の最上部に表示される
- [ ] 異常状態のコースには対処方法（ECマイページでのカード更新等）への導線が表示される（お支払い方法の変更は US-5-04 と同じ着地）
- [ ] 重要なお知らせ（決済不備等）はPUSH通知でも届く（US-5-02 の「重要なお知らせ」カテゴリ＝常時ON）
