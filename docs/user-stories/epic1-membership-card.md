# Epic 1: 会員証

> 会員であることの証明と、ランク（ステータス）体験の中核。アプリの第一利用シーン「店舗での会員証提示」を、会員とスタッフの両側から成立させる。

---

## US-1-00 既存会員としてログインする・機種変更で引き継ぐ

> **会員**として、いつものEC会員情報でアプリにログインし、機種変更しても同じように使いたい。それは新しい登録をやり直したくないし、自分のステータスを失いたくないからだ。

**受け入れ条件（DoD）**
- [ ] 既存のEC会員情報でログインでき、自分の氏名・ステータス・定期・履歴が表示される（※依存: OP-06 認証・ID連携方式）
- [ ] 機種変更後の端末で再ログインすると、同じ会員情報・ステータスに復帰する
- [ ] 未ログイン状態では、会員証QR・個人情報・購入情報は一切表示されない
- [ ] ログインに失敗した場合、原因（入力誤り/通信等）が分かるメッセージと再試行手段がある

## US-1-01 レジで素早く会員証を提示する

> **会員**として、レジで待たせずに会員証を提示したい。それは後ろのお客様を気にせず、スムーズに会計を済ませたいからだ。

**受け入れ条件（DoD）**
- [ ] ホームのファーストビューから1タップで会員証QRを表示できる（配置の詳細はデザイン工程で決定）
- [ ] アプリのコールドスタートから、**スキップ操作1回を含めて**QR表示まで3秒以内（自動計測: アイコンタップ起点〜QR描画完了）
- [ ] 起動演出中もスキップ・会員証ボタンが常時表示され、タップから0.3秒以内に反応する
- [ ] QR表示中は画面輝度が端末の最大値に引き上げられる
- [ ] QRには会員IDが含まれ、会員ランクは含まれない（ランクはスマレジ/EC側データを正とする）

## US-1-02 電波やシステムの都合に左右されず会員証を使う

> **会員**として、地下の店舗（GINZA SIX B1F等）で電波がなくても、システムメンテナンス中でも会員証を提示したい。それはレジの前で「表示できない」と慌てたくないからだ。

**受け入れ条件（DoD）**
- [ ] 機内モード・圏外の状態でも、QRコードが表示される（事前生成・キャッシュ）
- [ ] オフライン時に表示されるコードでも、店舗で会員特定が可能である（※依存: OP-01 トークン方式の採否。静的ID方式なら同一コード、トークン方式ならオフライン代替設計とセットで成立）
- [ ] サーバメンテナンス・API障害中も、会員証QRはキャッシュから表示できる
- [ ] オフライン・メンテ中はその旨が表示され、最新でない可能性のあるデータ（ポイント等）と区別できる

## US-1-03 自分のステータスと特典を確認する

> **会員**として、いまの自分のステータスと受けられる優待を確認したい。それは自分がどう遇されるのかを知り、BI-SUとの関係を実感したいからだ。

**受け入れ条件（DoD）**
- [ ] ホームに現在の公式ステータス（MEMBER / SILVER / GOLD / PLATINUM / DIAMOND）が英字表記で表示される
- [ ] 表示されるステータスは「前年1/1〜12/31の購入累計で確定した当年ステータス」である
- [ ] 現在の還元率（100円=Npt）が表示される
- [ ] ポイント残高が表示される（※依存: OP-05 ポイント残高API・有効期限の仕様）
- [ ] ランク別の優待内容（バースデーギフト・シーズナルギフト・イベントご招待）が静的一覧で確認できる（※依存: OP-10 特典表示の扱い）
- [ ] 画面に集計ルールの注記（前年実績で決まり当年1年間適用される旨）が表示される

## US-1-04 翌年ステータスへの進捗を確認する

> **会員**として、今年いくら購入していて、次のランクまであといくらかを知りたい。それは翌年のステータスを意識した買い物の楽しみがほしいからだ。

**受け入れ条件（DoD）**
- [ ] 今年（1/1〜現在）の購入累計額が表示される（※依存: OP-03残課題＝税込/税抜・返品減算等の集計細部）
- [ ] 次のランク閾値（30万/50万/100万/200万円）までの残額と進捗バーが表示される
- [ ] 進捗は「翌年ステータスの見込み」であることが分かる表現になっている（現ステータスとの混同を防ぐ）
- [ ] 当年累計が現ステータスの閾値未満でも、マイナス等の不自然な表示にならない
- [ ] DIAMOND会員には進捗の代わりに最上位への感謝メッセージが表示される

## US-1-05 ランクに応じた起動体験を楽しむ

> **会員**として、アプリを開くたびに自分のステータスにふさわしい特別な体験をしたい。それはBI-SUの世界（ボルネオの聖地）に近づいていく実感がほしいからだ。

**受け入れ条件（DoD）**
- [ ] 起動のたびに、ランク別のボルネオ・ズーム演出が再生される（MEMBER=上空→島 … DIAMOND=洞窟→巣）
- [ ] 演出はランクの上下にかかわらず、現在の公式ステータスに基づいて表示される
- [ ] スキップボタンが常設され、いつでも演出を飛ばせる。会員証ボタンへの直行時は演出を中断してよい
- [ ] 演出の着地ビジュアルがそのままホーム背景になり、体験が途切れない
- [ ] OSの「視差効果を減らす」設定時は演出が短縮される

## US-1-06 （スタッフ）会員IDをスマレジに取り込む

> **店舗スタッフ**として、会員のQRをその場で読み取って会員IDをスマレジに取り込みたい。それは入力の手間とミスをなくし、接客に集中したいからだ。

**受け入れ条件（DoD）**
- [ ] アプリが表示するQRを、スマレジ（または接続スキャナ）で読み取ると会員IDが転記される
- [ ] QRのコード体系がスマレジの読取仕様と一致している（※依存: スマレジ読取仕様の確認）
- [ ] 読み取り→会員特定までがレジ操作の流れの中で完結する（別端末・別アプリへの持ち替えがない）
- [ ] ※購入データの履歴・累計への反映は US-4-02 で受け入れる（本ストーリーは読み取り・転記まで）

## US-1-07 QRが使えない時に代替手段で本人確認する

> **会員**として、スマホの電池切れやアプリ不調のときでも会員として扱ってほしい。それはせっかく店舗まで来たのに優待を受けられないのは悲しいからだ。

**受け入れ条件（DoD）**
- [ ] QR画面から会員IDをテキストで確認できる（プライバシー配慮のためマスク表示＋タップで開示）
- [ ] QRの生成に失敗した場合、自動で会員IDテキスト＋バーコード等の代替表示に切り替わる
- [ ] スマホ自体を出せない場合の店頭照会手順（氏名＋電話番号等）が店舗オペレーションとして定義されている（※依存: 店舗手順書）

## US-1-08 年次のステータス切替わりを理解する

> **会員**として、年が変わって自分のステータスがどうなったかを正しく知りたい。それは新しい1年の優待と目標を、誤解なく把握したいからだ。

**受け入れ条件（DoD）**
- [ ] 1/1以降にアプリを開くと、新しい当年ステータスと、その根拠（前年の購入累計）が確認できる
- [ ] 進捗バー・当年累計は0円から再スタートし、前年の数値と混在しない
- [ ] ステータスが下がった場合も、否定的な文言を使わず、現ステータス準拠の演出・表示で迎える
- [ ] 切替時期（1/1）と適用期間が画面の注記で確認できる
