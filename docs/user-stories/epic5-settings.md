# Epic 5: 設定

> アカウント・通知の管理と、ブランドの世界観への入り口。変更系の操作は「アプリ=閲覧専用」の方針に基づき、ECマイページへ橋渡しして完結させる。

---

## US-5-01 アナツバメとブランドの物語を知る

> **会員**として、アナツバメとツバメの巣のことをもっと知りたい。それは自分が口にしているものの背景（ボルネオの洞窟・自然との共生）を理解して、安心と愛着を持ちたいからだ。

**受け入れ条件（DoD）**
- [ ] 「アナツバメについて」のコンテンツページがあり、アナツバメ・ボルネオの洞窟・巣の採取（ヒナが巣立った後の巣のみを採取するサステナビリティ）の物語が読める
- [ ] 起動演出と同一のキービジュアル・カラーパレットを使用し、ブランド責任者のレビュー承認を得ている
- [ ] 効能・効果を断定する表現を含まない（薬機法・景品表示法の制作チェックを通過している）
- [ ] オフラインでも閲覧できる（アプリ内蔵コンテンツ）

## US-5-02 PUSH通知の配信を設定する

> **会員**として、受け取る通知を自分で選びたい。それは必要なお知らせ（お届け予定など）は逃さず、不要な通知には煩わされたくないからだ。

**受け入れ条件（DoD）**
- [ ] 通知のカテゴリ別ON/OFF（お知らせ / お届け予定のリマインド / キャンペーン情報）が設定できる（※依存: OP-09 配信基盤の採否。配信を行わない決定の場合は本設定ごと非表示にする）
- [ ] 「重要なお知らせ」（決済不備等・US-3-05連携）は常時ONで、OFFにできないことが明示される
- [ ] 初期値: お知らせ=ON / お届けリマインド=ON / キャンペーン=OFF（※要決定: 初期値の最終確認）
- [ ] OSの通知許可が未許可の場合、その旨と端末設定への導線が表示される
- [ ] OFFにしたカテゴリの通知は配信されない（配信基盤側と連動する）

## US-5-03 登録情報（メールアドレス・住所・氏名）を変更する

> **会員**として、引っ越しや改姓、メール変更のときに登録情報を最新にしたい。それは定期便や大切な連絡を確実に受け取りたいからだ。

**受け入れ条件（DoD）**
- [ ] 設定から「メールアドレス変更」「住所変更」「氏名変更」それぞれの手続きに進める（ECマイページの該当ページへ遷移。遷移体験は共通DoD参照）
- [ ] メール: 変更完了が会員に確認できる（完了表示または確認メール）。アプリの表示メールアドレスが更新される
- [ ] 住所: 定期コースの次回お届けへの適用タイミングが案内される。変更後の届け先がアプリまたはEC側で確認できる
- [ ] 氏名: アプリのホーム・会員証・設定に表示される氏名が更新される

## US-5-04 お支払い方法を更新する

> **会員**として、カードの期限切れや変更のときに支払い方法を更新したい。それは定期便を止めずに続けたいからだ。

**受け入れ条件（DoD）**
- [ ] 設定からお支払い方法の変更手続きに進める（ECマイページへ遷移。※依存: OP-08）
- [ ] US-3-05（決済異常）の対処導線と同じ着地である
- [ ] アプリ内でカード情報を入力・保持せず、カード番号をフル表示しない

## US-5-05 問い合わせる

> **会員**として、困ったときにアプリから問い合わせたい。それは電話窓口を探し回らずに解決したいからだ。

**受け入れ条件（DoD）**
- [ ] 設定に「お問い合わせ」があり、窓口（フォーム/メール/電話等）に進める
- [ ] 受付時間・回答目安が表示される
- [ ] よくあるご質問（FAQ）が参照でき、自己解決を試せる

## US-5-06 ログアウトする

> **会員**として、共有端末や機種変更時にログアウトしたい。それは自分の情報を他人に見られたくないからだ。

**受け入れ条件（DoD）**
- [ ] 設定からログアウトでき、確認ダイアログを経て実行される
- [ ] ログアウト後、会員情報・QR・履歴等の個人情報が端末上で参照できない（キャッシュも消去）
- [ ] 再ログイン（US-1-00）で同じ会員情報に復帰できる

## US-5-07 アカウントを削除（退会）する

> **会員**として、利用をやめるときにアカウントを削除したい。それは自分のデータの扱いを自分で決めたいからだ。

**受け入れ条件（DoD）**
- [ ] アプリ内からアカウント削除の手続きに進める（App Store / Google Play の審査要件）
- [ ] 削除により何が消え、何が残るか（EC会員資格・定期契約・ポイントへの影響）が手続き前に明示される（※依存: EC側の退会仕様）
- [ ] 誤操作で削除されない確認ステップがある

## US-5-08 規約・プライバシーポリシーを確認する

> **会員**として、利用規約とプライバシーポリシーをいつでも確認したい。それは自分の情報がどう扱われるかを知っておきたいからだ。

**受け入れ条件（DoD）**
- [ ] 設定から利用規約・プライバシーポリシーが閲覧できる
- [ ] プライバシーポリシーに位置情報・PUSH通知・利用状況計測の利用目的が記載されている
- [ ] 文書の改定時に最新版が表示される
