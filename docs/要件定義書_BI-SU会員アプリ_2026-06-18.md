## 1. 背景・目的

- BI-SU（エムスタイルジャパンが運営する、人向けのツバメの巣 健康食品・化粧品ブランド）の**会員向けスマホアプリ**を作る。Claude Code の練習を兼ねたプロジェクト。✅
- 進め方は2段階: **①HTMLモックアップでデザイン・体験を確定 → ②Flutterで本実装**。✅
- アプリの中核コンセプトは「**会員ランクが上がるほど、ツバメの巣の源流（ボルネオの聖地）に近づいていく**」体験演出。✅

## 2. 対象ユーザー・利用シーン

- **ハイブリッド方式** 🆕v2.1: ログイン不要で基本機能が使える。連携は任意とし、連携時に会員機能を解放する。
  - **未登録ユーザー（非会員）**: アプリをダウンロードするだけで、仮会員証・店舗ポイント・チェックイン・ストアーズ・ブランドコンテンツを利用可能
  - **BI-SU の会員（EC購入者・店舗来店者）**: ログイン＋楽楽連携により、定期情報・購入履歴・ランク制度・全会員機能を利用
- 主な利用シーン: ①店舗でのバーコード提示（会員証/仮会員証）、②店舗ポイントの獲得・確認、③チェックイン（来店記録）、④定期購入の確認、⑤注文履歴の確認、⑥店舗を探して来店、⑦自分のランク・ポイントの確認。
- ⚠ 利用シーン①は**店舗側の読み取りオペレーション（誰が・何で読み・どう処理するか）とセット**で初めて成立する。アプリ単体では完結しない（→OP-01, OP-27）。

## 5. 非機能要件

| ID | 区分 | 要件 |
|---|---|---|
| NFR-01 | 対応OS | iOS / Android（Flutter）。💭仮案: iOS 16以上 / Android 10（API 29）以上 →OP-22で確定 |
| NFR-02 | 配布性（モック） | 単一HTML・オフラインで全機能動作（現行 約7.2MB）※本実装フェーズでは参考扱い |
| NFR-03 | 演出品質 | 起動演出は約5.5秒以内＋スキップ可。💭**起動からバーコード提示まで目標3秒以内**の導線（スキップ・バーコード直行）を確保（レジ前渋滞の防止。→OP-07）。計測: コールドスタートからバーコードモーダル表示完了までの実測時間 🔄v2.1 |
| NFR-04 | ブランド表現 | 公式トンマナ（アイボリー×シャンパンゴールド・明朝系）準拠。ランクカラーは公式準拠（PLATINUM=青系・DIAMOND=深紅系） |
| NFR-05 | 耐障害性・フォールバック | **Flutter（iOS 16+/Android 10+）前提のため、SVGフォールバックは不要**。代わりに以下のフォールバック条件を定義: ①**reduced-motion有効時** → 静止画（演出最終フレーム）で即ホーム遷移 ②**動画未ダウンロード時** → プレースホルダー画像＋プログレス表示 ③**バッテリー節約モード時** → 静止画フォールバック（動画再生を抑止）。💭本実装では**通信不可・API障害・メンテナンス時の画面挙動**も定義必須 |
| NFR-06 | アクセシビリティ | reduced-motion対応（上記NFR-05と連動）。本実装ではOSの文字サイズ設定への追従（200%まで）と、写真背景上のテキストコントラスト基準（WCAG AA目安）を追加（顧客層に高年齢層を含むため） |
| NFR-07 | データ鮮度 | 💭表示データの更新頻度と「いつ時点の情報か」の画面表示を定義（アプリ表示とEC・店頭レシートの正本宣言を含む。→OP-12） |
| NFR-08 | 画面対応 | 💭スマホ縦持ち優先。画面幅320〜430pt級で崩れないこと（モックは390×844基準） |
| **NFR-09** | **セキュリティ** 🔄v2.1 | 認証トークンのセキュアストレージ保存（Keychain/Keystore）。通信はすべてTLS 1.2以上。バーコードに個人情報を直接埋め込まない（🔄v2.1: QR→バーコード）。ログアウト時のローカルキャッシュ完全消去。バックグラウンド遷移時の画面マスク（個人情報保護）。**楽楽API認証の安全性確保**（APIキーの秘匿・通信経路の暗号化・アクセス権限の最小化）🆕v2.1 |

## 9. 制約・前提

- 練習プロジェクトのため、**会員・注文・距離・ポイント等の表示値はすべてダミー**（商品名・価格・店舗情報・ランク制度のみ公式掲載値を参照）。
- 公式サイトの商品画像・店舗画像・ロゴ画像を使用 → **社外に出す場合は権利確認が必要**（→OP-16）。
- AI生成動画の右下に透かしあり（本番前に除去要検討）。
- ロゴ3D形状は手作業トレース → 公式ベクターデータ入手後に差し替え推奨。
- モック専用要素（ランク切替UI・ダミー注記等）は本実装リリースビルドから除外する。
- **認証基盤はMIRANEST App（既存）と同等のアーキテクチャを前提とする**（認証コード送信のバックエンドを共用または同等設計で新設）。🔄v2.1: データ紐付け先をS3→**楽楽EC**に変更。楽楽APIとの連携基盤が必要。
- 🆕v2.1 **ハイブリッド方式**: ~~既存会員を前提~~ → ログイン不要で基本機能が使えるアプリ。非会員（未登録ユーザー）もアプリをダウンロードするだけで仮会員証・店舗ポイント・チェックイン等を利用可能。
- 🆕v2.1 **祇園店舗PR制約**: アプリ内テキスト・同意文言は祇園店舗のPR制約に準拠する必要がある（→OP-32）。
